# 🛡️ SISTEMA DE ALERTAS DE SEGURANÇA

## 📋 Visão Geral

O **Sistema de Alertas de Segurança** é uma área **exclusiva para capitães aprovados** que permite:

- 🔍 **Consultar alertas** de outros capitães sobre clientes problemáticos
- ⚠️ **Reportar incidentes** com clientes (não pagamento, comportamento inadequado, danos, etc.)
- 🤝 **Proteger a comunidade** profissional de pesca partilhando informações importantes
- 📊 **Visualizar estatísticas** de alertas por gravidade e período

---

## 🎯 Objetivo

Este sistema foi criado para:

1. **Proteger capitães** de clientes que já causaram problemas (burlas, não pagamento, danos)
2. **Partilhar experiências** entre profissionais de forma ética e responsável
3. **Criar uma base de dados confidencial** de incidentes verificados
4. **Permitir decisões informadas** antes de aceitar reservas

---

## 🔐 Acesso e Segurança

### Quem Pode Aceder?

✅ **Apenas capitães aprovados** pela administração  
❌ Clientes **NÃO têm acesso**  
❌ Capitães pendentes **NÃO têm acesso**  

### Como Aceder?

1. **Dashboard do Capitão** → Card "Alertas de Segurança" (vermelho com ícone 🛡️)
2. **URL direto**: `alertas-seguranca.html` (após login como capitão)

### Segurança e Responsabilidade

⚠️ **IMPORTANTE**: Este sistema é **confidencial** e deve ser usado com **responsabilidade**:

- ✅ Reporte apenas **incidentes reais e comprovados**
- ✅ Seja **objetivo e profissional** nas descrições
- ✅ Inclua **evidências e contexto**
- ❌ **NÃO** use para vingança pessoal
- ❌ **NÃO** reporte informações falsas
- ❌ **NÃO** partilhe estas informações fora da plataforma

🚨 **Uso indevido resulta em suspensão imediata da conta e possíveis ações legais.**

## 📊 Níveis de Gravidade

### 🟢 BAIXO
- Incidentes menores
- Atrasos, pequenos inconvenientes
- Cliente difícil mas não grave
- **Exemplo**: Chegou 1h atrasado, reclamações excessivas

### 🟡 MÉDIO
- Incidentes que requerem atenção
- Comportamento inadequado moderado
- **Exemplo**: Cancelamento de última hora, não seguiu instruções

### 🟠 ALTO
- Incidentes graves
- Danos ao barco/equipamento
- Comportamento muito inadequado
- **Exemplo**: Causou danos (€500+), embriaguez com danos

### 🔴 CRÍTICO
- **Evitar este cliente!**
- Burla, não pagamento
- Risco de segurança
- Comportamento perigoso/ameaçador
- **Exemplo**: Não pagou €1.500, agressão verbal, risco de vida

## 🏷️ Categorias de Incidentes

| Categoria | Descrição | Ícone |
|-----------|-----------|-------|
| **Não Pagamento / Burla** | Cliente não pagou ou tentou burla | 💸 |
| **Comportamento Inadequado** | Desrespeito, agressividade | 🚫 |
| **Danos ao Barco/Equipamento** | Quebrou ou danificou material | 🔧 |

| **Cancelamento de Última Hora** | Cancelou sem aviso prévio adequado | ⛔ |
| **Outro** | Outros tipos de incidentes | ⋯ |

## 📝 Como Reportar um Incidente

### Passo 1: Aceder ao Sistema
`Dashboard Capitão` → `Alertas de Segurança` → Tab **"Reportar Incidente"**

### Passo 2: Preencher o Formulário

#### 1️⃣ Selecionar Nível de Gravidade
- Escolha entre: Baixo, Médio, Alto ou Crítico
- **Dica**: Seja realista, não exagere

#### 2️⃣ Selecionar Categoria(s)
- Pode selecionar **múltiplas categorias**
- Exemplo: "Não Pagamento" + "Comportamento Inadequado"

#### 3️⃣ Dados do Cliente (Obrigatórios)
- Nome completo
- Email
- Telefone

#### 4️⃣ Descrição Detalhada (Mínimo 50 caracteres)
Inclua:
- **Data** do incidente
- **Local** / Tipo de saída
- **O que aconteceu** exatamente
- **Testemunhas** (se houver)
- **Evidências** (fotos, mensagens, etc.)
- **Tentativas de resolução**

**Exemplo de boa descrição:**
```
Cliente João Silva reservou saída de 2 dias para pesca ao atum 
(€1.200) em 15/03/2026. No primeiro dia apresentou-se em estado 
de embriaguez (testemunhas: tripulantes Miguel e Pedro). Causou 
danos em 2 canas profissionais (€400 cada, fotografias anexadas). 
No segundo dia recusou-se a pagar alegando que "não apanhou peixe 
suficiente". Tentei resolver amigavelmente via WhatsApp 
(prints guardados) mas cliente bloqueou-me. Polícia Marítima 
contactada (ocorrência n.º 2026-PM-1234).
```

#### 5️⃣ Data do Incidente
- Data em que o incidente ocorreu
- Máximo: hoje

#### 6️⃣ Declaração de Responsabilidade
- ✅ Concordar que as informações são **verdadeiras**
- ✅ Compreender as consequências de reportes falsos

### Passo 3: Enviar
- Clicar **"Enviar Alerta"**
- O alerta fica **imediatamente visível** para todos os capitães
- Você pode consultar na tab "Consultar Alertas"

3. **Filtro por Categoria**
   - Todas

   - Comportamento
   - Danos

   - Embriaguez
   - Cancelamento

📌 **Dados do Cliente**
- Nome completo
- Email
- Telefone

🏷️ **Detalhes do Incidente**
- Nível de gravidade (cor + badge)

- Reportado por qual capitão


### Armazenamento

- **LocalStorage**: `alertasSeguranca`
- Estrutura JSON com array de objetos

### Estrutura de Dados (Alerta)

```javascript
{
    id: 'ALT001',                        // ID único
    nomeCliente: 'João Silva',
    emailCliente: 'joao@email.com',
    telefoneCliente: '+351 912 345 678',
    gravidade: 'critico',                // baixo|medio|alto|critico
    categorias: ['nao-pagamento', 'comportamento'],
    descricao: 'Texto detalhado...',
    dataIncidente: '2026-03-05',
    dataReporte: '2026-03-06',
    reportadoPor: 'Cap. Pedro Costa',
    barco: 'Blue Marlin Hunter',
    verificado: false                     // Admin pode verificar
}
```

### Funcionalidades JavaScript

| Função | Descrição |
|--------|-----------|
| `verificarAutenticacaoCapitao()` | Verifica se usuário é capitão aprovado |
| `carregarAlertas(filtros)` | Carrega e filtra alertas |
| `renderizarAlertas(alertas)` | Renderiza cards de alertas |
| `atualizarEstatisticas()` | Atualiza contadores no topo |
| `configurarFormulario()` | Valida e submete novo alerta |
| `configurarFiltros()` | Aplica filtros em tempo real |

## 🔄 Fluxo Completo

```
1. Capitão acede ao dashboard
2. Clica em "Alertas de Segurança"
3. Sistema verifica se é capitão aprovado
4. Se não: redireciona com erro
5. Se sim: carrega alertas do localStorage
6. Capitão pode:
   a) Consultar alertas (filtrar, pesquisar)
   b) Reportar novo incidente (formulário completo)
7. Ao reportar:
   - Validação do formulário
   - Criação do alerta
   - Armazenamento em localStorage
   - Atualização de estatísticas
   - Feedback de sucesso
   - Redirect para tab "Consultar"
8. Alerta fica imediatamente visível para todos os capitães
```

## ⚖️ Considerações Legais e Éticas

### ✅ Uso Correto

- Proteger a comunidade profissional
- Partilhar experiências reais
- Prevenir futuros incidentes
- Documentar problemas comprovados

### ❌ Uso Incorreto

- Difamação
- Vingança pessoal
- Informações falsas
- Discriminação
- Partilha externa de dados

### 🔒 GDPR / Privacidade

⚠️ **NOTA IMPORTANTE**: Este sistema armazena dados pessoais (nome, email, telefone). Em produção:

1. **Consentimento**: Cliente deve ser informado de possível reporte
2. **Justificação**: Legítimo interesse na proteção da atividade comercial
3. **Acesso limitado**: Apenas capitães verificados
4. **Direito ao esquecimento**: Processo para remover alertas após resolução
5. **Verificação**: Admin deve verificar alertas graves
6. **Prazo**: Alertas podem expirar após X meses sem novas ocorrências
